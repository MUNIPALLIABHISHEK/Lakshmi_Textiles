01
🔍 Problem Statement

The Challenge at
Lakshmi Textile

Lakshmi Textile is an Indian textile shop struggling with zero digital tracking of inventory. All materials coming in and going out are managed through paper-based processes.

"We don't know how much stock we have, what's selling, or what we're losing."
— A common frustration in Indian textile retail
📝 Paper-Based Tracking
Manual registers that get lost, damaged, or have calculation errors
❌ No Stock Visibility
No real-time view of what's in the Godown vs. the shop floor
💸 Untracked Losses
Wastage, damage, and theft go completely undetected
🧮 Manual Calculations
Pricing, margins, and GST calculated by hand — prone to errors

💡 Our Solution

A Standalone Glassmorphic
Web Application

We built a fully offline, serverless web application that runs by simply opening an HTML file. No internet, no backend, no installation.

  • Runs in any browser — Chrome, Firefox, Edge
  • All data saved locally using Browser LocalStorage
  • Export & Import JSON backups with one click
  • Pre-seeded with sample Indian textile data

🏪 Feature — Billing

Dukaan Billing
(Counter POS)

A touchscreen-friendly Point of Sale interface for counter staff. Select category, pick item, set quantity, and bill — all in seconds.

1. Select Category
Tap visual tiles: Saree, Kurta, Than, Lungi, Dupatta, Accessories
2. Choose Item
Auto-filtered dropdown showing only items with available stock
3. Set Quantity
Large + / − buttons for meters or pieces. Supports decimals (0.5m).
4. Complete Sale
Bill auto-calculates subtotal + 5% GST. Stock deducted from Dukaan instantly.

🧮 Feature — Calculators

Smart Textile
Business Calculators

Four purpose-built calculators that solve daily math problems for textile shop owners.

💰
Kamaai Calculator (Profit Margin)
Input: Buy Price + Overhead + Margin %
Output: Selling Price, Net Profit (₹), Markup %, GST-inclusive Price
Interactive slider for margin % that updates results in real time.
📏
Than Yield (GSM to Meters)
Input: Roll weight (Kg), fabric width (cm), GSM
Output: Estimated length in meters + yard conversion
Verify if suppliers sent correct fabric length without unrolling!
🌸
Mausam Predictor (Seasonal Demand)
Input: Last month sales + Season dropdown (Diwali +80%, Monsoon -25%)
Output: Recommended order quantity with safety buffer
9 pre-set seasons from Indian festivals to school reopening.
⚠️
Kharaab Maal (Wastage Loss)
Input: Wasted meters + Cost per meter + Monthly sales
Output: Total loss (₹), Loss as % of sales, Severity gauge
Visual SVG gauge shows how badly wastage is eating into profits.

💾 Architecture

Data Architecture
& Backup System

All data is stored in the browser using localStorage with structured JSON. The app initializes with rich seed data on first launch.

Data Modules
📦 Stock Items15 seeded
🤝 Suppliers5 seeded
🚚 Orders5 seeded
🛒 Sales Records10 seeded
⚠️ Wastage Log2 seeded
📥 One-Click Backup Export
Downloads a complete JSON file with all stock, sales, supplier, and order data. Named with today's date for easy tracking.
📤 Drag & Drop Restore
Upload a backup JSON file to restore all data — transfers between computers or recovers after browser history clearing.
🔒 Session Management
Login sessions use sessionStorage (auto-logout on browser close). Data persists in localStorage across sessions.
